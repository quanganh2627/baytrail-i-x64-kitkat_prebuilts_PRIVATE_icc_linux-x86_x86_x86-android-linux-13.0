Intel® C++ Compiler v13.0 for Android * OS is a cross-compiler for software developers to create C and C++ applications on host Linux* operating systems targeting for the IA-32 architecture-based Android* operating system.

You can develop applications for IA-32 systems running on Android* OS in one of two ways:

  • Using the Intel C++ Compiler as a stand-alone tool
  • Enabling the Intel toolchain within the Android NDK build system

The following sections provide general guidelines for both methods.

Using the Intel® C++ Compiler as a Stand-Alone Tool

The Intel C++ Compiler requires a GNU compiler toolchain for the Android OS to work.

You can fulfill the requirement in one of two ways:

  • Install the Android NDK for 32-bit Linux* OS from http://developer.android.com/sdk/ndk/index.html (The link refers to the latest version of NDK - r8e.)
  • Use the GNU toolchain which is included in the Android source repository

Setting Environment Variables

You must set the environment variables to point the Intel C++ Compiler to the the GNU toolchain. The correct method depends on whether you use the NDK or the Android* Open Source Project (AOSP) environment.

Using the NDK environment:

export NDK= #set to the NDK root directory (example: /work/android-ndk-r8e/)

export ANDROID_SYSROOT=$NDK/platforms/android-14/arch-x86                     
export ANDROID_GNU_X86_TOOLCHAIN=$NDK/toolchains/x86-4.6/prebuilt/linux-x86

Using the AOSP environment:

export TOPDIR= #set to the AOSP root directory (example: /export/JB_WW11)

export ANDROID_SYSROOT=$TOPDIR/prebuilts/ndk/8/platforms/android-14/arch-x86       
export ANDROID_GNU_X86_TOOLCHAIN=$TOPDIR/prebuilts/gcc/linux-x86/x86/i686-linux-android-4.6

Setting PATH to Intel Compiler Binaries

You must set the PATH environment variable to include a path to Intel C++ Compiler binaries. One method for setting the environment variables quickly is to source the environment script, as shown below:

source /opt/intel/CCAndroid13.0.0.XXX/bin/iccvars.sh

Enabling the Intel toolchain within the Android NDK Build System

Follow these steps to enable the ICC toolchain inside the NDK build system:

  1. Install Intel C++ Compiler

  2. Set NDK environment variable, for example:
    NDK=/work/android-ndk-r8e

  3. Invoke the script integrate_into_ndk.sh:
    source /opt/intel/CCAndroid13.0.0.XXX/bin/integrate_into_ndk.sh

Building a Sample Application with the Intel® C++ Compiler for Android* OS

To build a sample application with the Intel compiler, you need to specify the Intel compiler toolchain in the build command. Use the NDK_TOOLCHAIN option and specify "icc", for example:

NDK_TOOLCHAIN=icc

 

Here is an example of how to build the application:

  1. Build the sample hello-jni application from NDK samples:

    cd $NDK/samples/hello-jni 
    ndk-build V=1 -B NDK_TOOLCHAIN=icc APP_ABI=x86 #

    where:

    V=1 specifies output command

    -B means rebuild all

    APP_ABI indicates target architecture

  2. Check for errors in the output.

Running Applications Built with ICC on a Target Device

The Intel C++ Compiler comes with libraries to support math operations and CPU dispatching. Every binary compiled with the Intel compiler must be able to access these libraries.

One way to provide the access is to add the following line to the jni/Android.mk file for your application:

LOCAL_LDFLAGS := -static-intel

Building an Application for the Release

By default, NDK is configured to build an application for debugging, and all optimizations are turned off. When the code is ready for deployment, you need to compile it for performance.

The first step is switching to the release mode. Add the following line to the Application.mk file:

APP_OPTIM:=release

The second step is compiling for a specific target processor. For example, to build application to run on the Intel® Atom™ processor, add the following line to the Application.mk file:

APP_CFLAGS:=-xSSSE3_ATOM

The third step is enabling advanced optimizations. For example, to increase the optimization level, add the following line to the Application.mk file:

APP_CFLAGS:=-xSSSE3_ATOM –O3

To enable interprocedural optimizations, add the –ipo option to APP_CFLAGS:

APP_CFLAGS:=-xSSSE3_ATOM –O3 –ipo

APP_OPTTIM and APP_CFLAGS can be given as arguments to the ndk-build command. For example:

ndk-build V=1 -B NDK_TOOLCHAIN=icc APP_ABI=x86 APP_OPTIM:=release APP_CFLAGS=”-xSSSE3_ATOM –O3”

For further details, refer to the “High-Level Optimizations (HLO)” and “Interprocedural Optimization (IPO)” sections in the Intel® C++ Compiler XE 13.0 User and Reference Guides supplied with the package.

Using Intel® C++ Compiler for Android* OS with Eclipse*

To use the Intel® C++ Compiler for Android* OS for building NDK based applications in Eclipse, you must:

  1. Install and integrate the Intel® C++ Compiler for Android* OS into the Android NDK as described in the previous sections.
  2. Install Android Development Tools (ADT) 20.0 or newer into Eclipse from http://developer.android.com/tools/sdk/eclipse-adt.html.
  3. Configure the Eclipse project.

You must do the following to configure the Eclipse project:

  1. Add Native Support to your existing Android project. Right click on your Android project ->Android Tools -> Add Native Support:

  2. Open the properties of an existing Android project. To view the properties dialog, open the Project menu in Eclipse and click on Properties:

  3. Navigate to the C/C++ Build -> Environment option:

    4.  

  4. Add the following environment variables:

     

    NDK= # set NDK root (example: NDK=/export/android-ndk-r8)
    ANDROID_SYSROOT="${NDK}/platforms/android-14/arch-x86"
    ANDROID_GNU_X86_TOOLCHAIN="${NDK}/toolchains/x86-4.6/prebuilt/linux-x86"

    5.  

  5. Add the following line to the jni/Application.mk file:

     

    APP_ABI := x86
    NDK_TOOLCHAIN := icc

    6.  

  6. Open the Project menu and click on Clean.

    7.  

  7. Click on Build or Run to use ICC to build the native parts of the Android project.

See Using Eclipse section in the Intel® C++ Compiler XE 13.0 User and Reference Guides supplied with the package for information on using the Intel(R) C++ Compiler in Eclipse* on Linux* OS.

Using PGO to Improve Android* OS Application Performance

  1. Add the following options to the CFlags in the jni/Android.mk file:

     

    LOCAL_CFLAGS := -prof-gen -prof-dir /sdcard

    2.  

  2. Add the WRITE_EXTERNAL_STORAGE permission to allow the application writing the PGO output to the sdcard folder. Add the following line to the AndroidManifest.xml file:

     

    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />

      

  3. Ensure that the profiling data gets written. The profiling data is written in default mode only if the application exits. Usually an application on Android* does not exit. Use the following options to enforce the application exit or to workaround this issue:
    1. Option 1: Call exit from JAVA code:
       

      System.exit(0);

       

    2. Option 2: Explicitly dump PGO data from the native code:
       

      #include <pgouser.h>
      _PGOPTI_Prof_Dump_All();

       

    3. Option 3: Use environment variable to write the performance data regularly while the application is running. To make the environment variables available for all applications, add them to the init.rc file of the Android Image:
       

      export INTEL_PROF_DUMP_INTERVAL 5000
      export INTEL_PROF_DUMP_CUMULATIVE 1

    NOTE
    The value INTEL_PROF_DUMP_INTERVAL is measured in microsecond and should not be greater than INT_MAX.

  4. Copy the generated dyn files from the target to the host in the source directory of the application:

     

    adb pull ...

    5.  

  5. Change the CFLAGS in Android.mk file to use the generated dyn files. You can use the -prof-dir parameter to point to a different file location.
     

    LOCAL_CFLAGS    := -prof-use

    6.  

See Profile-Guided Optimizations section in the Intel® C++ Compiler XE 13.0 User and Reference Guides supplied with the package for information on using PGO for optimizing applications built with the Intel(R) C++ Compiler on Linux* and Windows* operating systems.
